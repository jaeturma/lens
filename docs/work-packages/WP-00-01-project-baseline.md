# WP-00-01 — Project Baseline Inspection

## Status

Ready

## Objective

Document the existing Laravel 13 and Flutter project structure without changing application behavior.

## Scope

- Confirm Laravel and PHP versions
- Identify authentication approach
- Identify API route structure
- Identify existing tests and quality tools
- Locate or confirm the Flutter application
- Identify Flutter state management, HTTP client, routing, and secure storage
- Record existing conventions in the project documents

## Out of Scope

- Installing packages
- Creating modules
- Refactoring source code
- Changing database schema

## Acceptance Criteria

- [ ] Current backend stack is documented
- [ ] Current mobile stack is documented or marked not yet created
- [ ] Authentication status is documented
- [ ] Test and analysis commands are documented
- [ ] No application behavior is changed

## Definition of Done

- [ ] `docs/PROJECT-BASELINE.md` is created
- [ ] Only documentation files are changed
- [ ] Findings are concise and evidence-based
