# Testing Rule

- Add tests for confirmed behavior, permissions, validation, and failure cases.
- Prefer targeted tests while implementing one work package.
- Do not rewrite unrelated tests merely to make the suite pass.
- When a test fails, identify whether the defect is in implementation, fixture, environment, or test expectation.
- Report skipped or unexecuted tests honestly.
- Run the full suite only at epic, phase, or release checkpoints unless specifically requested.
